개인프로젝트

메인 화면에 2개의 버튼을 두어서 게시판에 입장 할 것인지, 회원 관리에 입장할 것인지 선택하고 각 버튼별 페이지 링크
각 페이지 마다 홈, 게시판, 회원관리로 이동하는 형태
HTML을 3개 사용해서 연결하는 방법 찾기

1. 게시판 형태
  1 게시판 목록 표현

    1-1 게시판 목록 순서는 최근글이 제일 위로 (O)
      1-1.a 가능하다면 목록 순서 선택지를 드랍 다운 형식으로 넣고 값에 맞게 순서를 재배치하여 표현 (O)
        -> 게시판 목록은 순서 정렬은 최신순, 오래된 순정도로만 필요한 기능이라 이 기능은 회원관리에서 표현하기로 함
        그 대신 검색을 통해 원하는 게시글 목록을 불러오는 식으로 표현
        -> 차후 게시판 데이터베이스에 시퀀스를 이용한 조회수를 넣어보고 글번호 정렬과 조회수 정렬을 넣어 보기

      1-1.b 게시글 목록에서 표현은 글번호, 글제목, 글내용(한글기준 20자 제한, 20자 초과시 ...으로 줄이기), 작성일시(서버시간 자동입력) (O)
        -> 목록에서는 글번호, 글제목(글자수 7자 제한), 작성자(글자 수 7자 제한), 작성일시(서버시간 자동입력)으로 수정


    1-2 게시판 목록을 누르면 글 내용 표현 (O)
      1-2.a 게시글 글 내용 래이아웃은 상단에 타이틀, 작성자, 중간에 내용이 있는 칸 표시 (O)
        1.2.a1 게시글 내용이 길 때, 내용 칸만 드래그기능을 넣고 내용을 볼 수 있게 구현해보기 (O)


    1-3 게시판 페이징을 넣어서 한 페이지에 글 10개씩 표현 (O)
      1-3.a 가능하다면 한 페이지에 글이 10개, 15개, 20개씩 표현되는 드랍다운 형식으로 기능 만들기 (O)


    1-4 게시판 목록에서 SPA형식에 맞춰서 글 작성 표현하기 (O)
      1-4.a display : none을 이용하고 글쓰기 버튼을 눌렀을 시 표현하는 형태로 구현 (O)

      1-4.b 글쓰기 완료를 누르면 데이터베이스에 데이터를 넣고 목록에 추가하는 기능 (O)
        -> 프론트엔드에서 목록만 추가하는 형태는 여러 유저가 이용하는 게시판 특성 상 
        다수가 글을 올려도 자신의 화면에는 자신이 올린 글만 보일 가능성이 있기 때문에
        쿼리를 이용해 새로 커밋된 데이터를 fetch로 받아서 그리는 방식을 채택 (SPA 방식 유지)


2. 회원관리 (O)
 2 게시판 목록 표현

    2-1 회원관리 목록 순서는 가입일 순서로 -> 시퀀스로 primary key에 순서 할당 했으니 primary key로 배열  (O)
      2-1.a 가능하다면 목록 순서를 클릭 버튼을 2개 만들어서 클릭하면 그 값에 맞게 순서를 재배치하여 표현 (O)


    2-2 회원 목록은 회원번호, 회원 아이디, 회원 비밀번호, 회원이름, 회원 전화번호 (O)


    2-3 검색기능을 넣어서 회원 정보를 검색 할 수 있게 만들기 - (X)
      2-3.a 검색기능을 세분화해서 번호, 아이디, 이름, 전화번호를 검색할 수 있게 할건지
      또는 통합 검색으로 뭘 검색하든지 결과에 걸리도록 할 것인지 가능하면 해보기
      -> 검색기능을 게시판에서 구현했기때문에 또 구현하는 건 프로젝트 의도와 어긋난다고 판단해서 이 기능의 우선순위는 나중으로


    2-4 회원가입 페이지는 따로 만들어서 데이터베이스에 넣는 역할을 하고 회원 관리 페이지는 목록만 보는 형식으로 (O)
